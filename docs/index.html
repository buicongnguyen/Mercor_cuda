Study Goal

Revision material for the Mercor GPU kernel optimization expert assessment. The role is about improving kernel performance using profiler-guided analysis — reading metrics such as L2 cache hit rate, L2 throughput, occupancy, and warp-stall reasons, deciding what they mean, and knowing when a metric is not useful. The material pairs definitions and core-concept notes with a bank of multiple-choice practice questions and explanations. The final mock adds a one-submission candidate flow and evidence-based self-evaluation so you can rehearse the unfamiliar-kernel work under pressure before taking the real assessment.

Execution

SIMT Architecture

SMs, warps, blocks, grids, warp schedulers, divergence, occupancy limiters, and latency hiding — the machine model every optimization reasons against.

Memory

Memory Hierarchy

Registers → shared → L1 → L2 → DRAM, coalescing and sectors, bank conflicts, vectorized loads, async copy, and read-only paths.

Profiling

Nsight Compute Metrics

Speed-of-light, achieved vs theoretical occupancy, L2 hit rate and throughput, sectors per request, warp-stall reasons, roofline — and the traps in each.

Techniques

Optimization Playbook

Tiling, fusion, ILP and unrolling, grid-stride loops, warp shuffles, tensor cores, inline PTX, CUB/Thrust/libcu++, streams and graphs.

Language

C++17 & Portability

C++17 features that matter in kernel code, plus CUDA ↔ HIP ↔ compute-shader mapping (Slang/HLSL/GLSL) and Blackwell-generation notes.

Judgment

Case Studies

Principal-level drills: diagnose full Nsight Compute reports, spot the bug in kernels you didn't write, review a claimed 4× speedup, triage a register-cap regression, and know when tuning is finished.

Practice format

MCQ Drills

116 questions in four formats — single answer, select-all-that-apply, arrange-in-order chains, and fill-in-the-blank code/command completions — many built around embedded profiler dumps and kernel code, with sourced production scenarios, explanations, per-section scores, and reshuffling retry. Works offline.

Assessment rehearsal

Six-Task Mock

An original, unofficial 100-point simulation with timed or untimed modes, autosave, final-submission lock, CUDA safety caps, evaluator rubrics, remediation links, and retake support.

Reasoning

CUDA Thinking Atlas

Visible flowcharts for constructing an answer, assigning data ownership, selecting synchronization scope, tracing block-wide sequences, debugging unfamiliar kernels, and turning profiler counters into falsifiable claims.
