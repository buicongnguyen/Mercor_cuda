Study Goal

Revision material for the Mercor GPU kernel optimization expert assessment. The role is about improving kernel performance using profiler-guided analysis — reading metrics such as L2 cache hit rate, L2 throughput, occupancy, and warp-stall reasons, deciding what they mean, and knowing when a metric is not useful. The material pairs definitions and core-concept notes with a bank of multiple-choice practice questions and explanations.

Execution

SIMT Architecture

SMs, warps, blocks, grids, warp schedulers, divergence, occupancy limiters, and latency hiding — the machine model every optimization reasons against.

Memory

Memory Hierarchy

Registers → shared → L1 → L2 → DRAM, coalescing and sectors, bank conflicts, vectorized loads, async copy, and read-only paths.

Profiling

Nsight Compute Metrics

Speed-of-light, achieved vs theoretical occupancy, L2 hit rate and throughput, sectors per request, warp-stall reasons, roofline — and the traps in each.

Techniques

Optimization Playbook

Tiling, fusion, ILP and unrolling, grid-stride loops, warp shuffles, tensor cores, inline PTX, CUB/Thrust/libcu++, streams and graphs.

Language

C++17 & Portability

C++17 features that matter in kernel code, plus CUDA ↔ HIP ↔ compute-shader mapping (Slang/HLSL/GLSL) and Blackwell-generation notes.

Practice format

MCQ Drills

72 questions in three formats — single answer, select-all-that-apply, and arrange-in-order chains — with explanations, per-section scores, a show-all toggle, and reshuffling retry. Works offline.
